Part 1 - Understanding the Python Interpreter

Before we jump into writing code, let's take a quick look at how Python works behind the scenes. This will help you understand what happens when you run your programs. The key player here is the Python Interpreter, the program that reads and executes your Python code.

On my Windows machine, for example, the interpreter is located at:

C:\Users\kodopy\AppData\Local\Programs\Python\Python313\python.exe

When you write Python code, you'll save it in a file with a .py extension (e.g., script.py). This extension tells your operating system that it's a Python file and allows the interpreter to recognize it. To run this file, you'll use a terminal (like Command Prompt on Windows or Terminal on macOS/Linux) and provide two pieces of information: the path to the Python interpreter and the path to your .py file.

Here's an example command:

C:\Users\kodopy> python C:\script.py

Let's break this command into its two main parts:

  1. python

    2. This is the command to launch the Python interpreter. If your system's environment variables (like PATH) include the directory where python.exe is stored, typing python is enough to start it. If this doesn't work on your machine, it means the path isn't set up yet. See the Software Section for instructions on how to fix this. Essentially, this part just tells your computer to run the Python program.

  2. C:\script.py

    3. This is the path to your Python file. After launching the interpreter with python, you pass the file path as an argument. The interpreter then locates your file and starts executing the code inside it.
