Git

Here are some common git commands.

Command Description
git init Initialize a new Git repository
git clone "url" Clone a repository from a remote source
git add "file" Stage changes for commit
git commit -m "message" Commit staged changes with a message
git status Show the status of the working directory
git log View commit history
git branch List branches or create a new one
git checkout "branch" Switch to another branch
git switch "branch" Alternative to checkout for switching branches
git merge "branch" Merge a branch into the current one
git pull Fetch and merge changes from a remote repository
git push Push local commits to a remote repository
git fetch Retrieve updates from a remote repository without merging
git rebase "branch" Apply commits on top of another branch
git reset "file" Unstage changes or reset commits
git stash Temporarily save uncommitted changes
git tag "name" Create a tag for a specific commit

GitHub

As always, the official GitHub Docs is the best resource to explore all topics in depth.

GitHub is an online service that allows you to host your repositories on a remote server. It offers both free and paid plans, with the free version offering plenty of features to work without any real hiccups.

While you work locally on your computer, the Git application tracks all changes in your repository. Once you're ready to push your updates, you can send them to GitHub, keeping your branch in sync. But GitHub is much more than just a remote backup of your project.

GitHub allows multiple people to work on the same project simultaneously. It includes a fully programmable Actions system to automate tasks based on specific events. You can also create REST APIs for your repositories, edit and view code online from anywhere, and safeguard your project against hard drive failures.

Even with the free version, you can create both public and private repositories. And now, GitHub offers Copilot, an AI-powered coding companion that helps boost productivity as you code.

Additionally, you can turn a repository into a website, just like this KodoPy Docs site! If you're interested, check out the GitHub Pages Guide.

The list of features goes on, and GitHub is always evolving. It may take time to explore everything it can do, but just try to not get stuck in all the minute details. Instead, learn what exists and what the features can do (high-level), and only get into the nitty-gritty details when you need it for your project. Chances are, you wont need every possible feature, so, just take a peek and move on if you don't need it! Just like speed shopping!

GitHub Desktop

GitHub Desktop provides a graphical user interface (GUI) to perform common Git operations, such as cloning repositories, committing changes, managing branches, and syncing with remote repositories, without needing to memorize or type Git commands. I would say that using the GUI version of Git seems like cheating or something but it isn't. If you like it, use it, thats why it's there. It is NOT tailored to beginners, a lot of professionals use it because it's fast, powerful, and lets you see everything that is happening. (Not trying to undermine Git and the command line, for me all tools have their place and purpose)
